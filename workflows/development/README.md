# Development Workflows

Workflows for code analysis, refactoring, and development best practices.

## Available Workflows

| Workflow | Description |
|----------|-------------|
| [api-compatibility-check](./api-compatibility-check.ds.md) | Check if a library or service API supports a specific method, endpoint, or capability |
| [architecture-patterns](./architecture-patterns.ds.md) | Identify inconsistent patterns, duplicate approaches, and architectural drift in the codebase |
| [config-schema-check](./config-schema-check.ds.md) | Check if a configuration system supports a specific option or setting |
| [dependency-capability-check](./dependency-capability-check.ds.md) | Check if a project's dependencies support a specific feature or version requirement |
| [test-coverage-gap-analysis](./test-coverage-gap-analysis.ds.md) | Analyze test files against project structure to identify untested code paths, functions, and modules |
